Mon site web : CV

Ce premier projet est mon CV que j'ai réalisé sous la forme d'un site web et que vous visitez actuellement.

Pour ce projet j'ai réalisé un fichier HTML pour la page principale qui contient mes informations ainsi que mon CV, et un deuxième fichier HTML qui correspond à la page de mon portfolio (la page actuelle).
Par la suite, j'ai fait un fichier CSS pour la mise en forme de mon site. Celui-ci permet d'ajouter de la couleur, ainsi qu'une architecture à mon site. De plus, par le biais de ce fichier CSS, j'ai pu adapter mon site pour un utilisateur sous smartphone (peut-être vous ?).

J'ai mes fichiers HTML, mon fichier CSS, et également un dossier spécifique dans lequel je vais piocher mes images. De plus, pour héberger mon site web, j'ai utilisé Github qui est un gestionnaire de version mais qui propose aussi la mise en page (et c'est pour cela que mon adresse contient "github.io").

Application mobile : Mareu

Mareu est un des projets que j'ai réalisé dans le cadre de ma formation chez OpenClassroom

Mise en situation :
Une entreprise en pleine croissance a de plus en plus de mal à gérer son planning pour les salles de réunion. Chaque réunion est planifiée en plaçant un post-it sur la porte de la salle en question.

Notre rôle :
Créer une application permettant de gérer les différentes réunions.
Ajouter ou supprimer une réunion.
Filtrer les réunions, par salle ou par date.
Faire en sorte que l'application puisse être affichée sur tous types d'écran, en portrait ou en paysage.

Application mobile : Todoc

Todoc fait aussi parti d’un des projets à traiter lors de ma formation chez OpenClassroom.

Mise en situation :
Une agence de nettoyage de locaux commerciaux a fait réaliser une application de gestion de tâches pour faciliter le travail de ses collaborateurs. Cependant, son prestataire a mis la clé sous la porte avant d’avoir pu réaliser la persistance des données sur l'application

Notre rôle:
Finir le travail de l'ancien prestataire en ajoutant une base de données à l'application existante.
Réaliser un MPD (Model Physique de Données), réaliser un diagramme d'utilisation et de classe.
Créer un fichier APK signé de l'application.

Application mobile : Go4Lunch

Go4Lunch fait aussi parti d’un des projets à traiter lors de ma formation chez OpenClassroom.

Mise en situation :
Au sein d'une entreprise, Go4Lunch sera une application utilisée par les employés. Elle permettra de rechercher un restaurant dans les environs, puis de sélectionner celui de son choix et en faire part à ses collègues.

Les differentes Fonctionnalité:
Système d'authentification par Facebook ou Google (Firebase).
Menu latéral, avec les informations de l'utilisateur.
Carte des restaurants aux alentours.
Liste des restaurants avec leurs informations.
Liste des collègues avec le choix de leurs restaurants.
Fiche détaillée d'un restaurant.
Système de like d'un restaurant.
Fonctionnalité de recherche.
Système de notification.
Application disponible en Anglais et Français.
Systeme de chat pour dialoguer avec les collègues.
